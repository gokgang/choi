JavaScript 문법(Syntax)

JavaScript 구문 규칙

    var x, y, z;
    x = 5; y = 6;
    z = x + y;

  // 변수 선언
  // 실효값 할당
  // 변수값 계산 및 결과값 할당

Values

JavaScript는 다음 두 가지 유형의 값을 정의한다.

  • 고정 값 : 실효값(Literals)
  • 변수 값 : 변수(Variables)

고정 값(Literals)

고정 값에 대한 두 가지 구문 규칙

1. 숫자는 정수와 실수를 구분하지 않는다.

    10.50
    1001

2. 문자열은 큰 따옴표(" ") 나 작은 따옴표(' ')로 묶어야 한다.

    "John Bidden"
    'John Bidden'
    '10.50'
    "1001"

변수(Variables)

Programming 언어에서 변수는 데이터의 값을 저장하는 데 사용된다.

JavaScript는 var 또는 let 지시어를 사용하여 변수를 선언한다.

변수란 컴퓨터의 기억장치에 실효값을 저장할 수 있는 곳의 주소를 개발자가 이름으로 정한 것을 말한다.

다음의 예에서 x는 변수로 선언되고 선언된 변수 x에 값을 할당한다.

    var x;
    x = 6;
    document.write(x);

연산자(Operator)

산술 연산자( + - * / )

  document.write( (5 + 6) * 10 );

할당 연산자( = ) : 변수 값 할당

    var x, y;
    x = 5;
    y = 6;
    document.write(x + y);

표현식(Expressions)

식은 값, 변수 및 연산자의 조합으로 값을 연산하며, 이를 평가한다고도 한다.

예를 들어 (5 * 10)은 50으로 평가된다.

  document.write(5 * 10);

연산식에는 변수도 포함될 수 있다.

    var x = 5;
    document.write(x * 10);

값은 숫자(Literals) 및 문자열(Strings)과 같은 다양한 유형이 될 수 있다.

예를 들어 "단군" + " " + "조선"는 "단군 조선"으로 평가된다.

  document.write("단군" + " " + "조선");

지시어(Keywords)

JavaScript에서 지시어는 브라우저가 수행할 작업을 명령한다.

지시어 var은 브라우저에게 지시어 다음에 있는 이름을 가진 메모리 공간을 확보할 것을 명령한다.

    var x, y;
    x = 5 + 6;
    y = x * 10;
    document.write(y);

설명문(Comments)

JavaScript code에 있는 설명문(Comments)은 무시되며 실행되지 않는다.

설명문은 주석이라고도 하며 한줄에서 Double Slashes( // ) 다음에 있는 문장은 실행되지 않는다.

또한 여러 줄을 한꺼번에 블록 주석 처리를 하려면 /* 와 */ 사이에 문자이 있게 한다.

  var x = 5;  // 이 문장은 실행된다.
  // var x = 6;  이 문장은 실행되지 않는다.

식별자(Identifier)

JavaScript에서 식별자는 변수 및 지시어, 함수, 레이블 등의 이름을 말한다. 식별자에 대한 규칙은 대부분의 프로그래밍 언어에서 같다. JavaScript에서 식별자는 반드시 첫 번째 문자는 문자, 밑줄( _ ) 또는 달러 기호($)이어야 한다. 이어지는 문자는 문자, 숫자, 밑줄 또는 달러 기호일 수 있다. 한글을 식별자로 사용할 수는 있으나 한글은 그대로 사용되는 것이 아니라 모두 유니코드로 변환하여 전송되므로 개발자의 컴퓨터에서는 제대로 실행되지만 전송과정에서 문제가 발생할 수 있다. 이는 파일 이름 역시 마찬가지이다. 한글로 파일 이름을 만들고 서버에 로드한 후 다시 다운로드해 보면 파일이름이 망가져 있는 것을 종종 볼 수 있다.

첫 번째 문자는 숫자를 사용할 수 없으며, 당연히 각종 연산자로 사용되는 기호는 식별자에 포함될 수 없다.

JavaScript는 대-소문자를 구분한다.

모든 식별자는 대-소문자를 구분한다.

변수 lastName 과 lastname는 두 개의 다른 변수이다.

    var lastname, lastName;
    lastname = 단군;
    lastName = 조선;
    document.write(lastname + " " + lastName);

JavaScript는 VAR 또는 Var 로 코딩하게 되면 지시어 var로 인식하지 않는다.

JavaScript의 변수 이름

예전부터 프로그래머는 여러가지 단어를 하나의 변수 이름으로 결합하는 다양한 방법을 사용했다.

그 예를 보면

  • Hyphen( - )으로 이름 결합(파일이름에서 주로 사용)

    • 예: first-name, last-name, master-card, inter-city

    Hyphen은 JavaScript에서 산술 연산자이므로 변수 이름으로 사용할 수 없다.

  • Underscore( _ )로 이름 결합

    • 예: first_name, last_name, master_card, inter_city

  • Upper Camel Case(Pascal Case)

    • 예: firstName, lastName, masterCard, interCity

    JavaScript의 사용 문자(Character Set)

    JavaScript는 Unicode(UTF-8) 문자를 사용한다.

    Unicode는 세계의 거의 모든 문자, 구두점 및 기호를 포함한다.

    Unicode의 처음 128 글자는 ASCII 문자와 같은 이진값(Binary)을 가지므로 HTML의 본문은 한글을 사용하더라도 JavaScript 변수 이름을 영문자와 숫자 만으로 작성하는 것은 호환성을 높이는 좋은 방법이다.

    HTML Unicode(UTF-8) 참조

    HTML 4는 UTF-8 을 지원하며, HTML 5는 UTF-8 과 UTF-16 을 모두 지원한다. UTF-8 은 1byte 8bits 문자 체계이며, UTF-16 은 2bytes 16bits 문자 체계이다.

    HTML 5의 표준은 Unicode UTF-8 이다.
