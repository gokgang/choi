HTML 요소

HTML 요소는 시작 매듭(Start Tag), 내용(Contents), 끝 매듭(End Tag)의 묶음으로 정의한다.

HTML 요소의 구문(Syntax)

<tagname> 문서 내용 ... </tagname>

예 :

<h1>문서의 주 제목</h1>
<p>이 글은 하나의 예문이다.</p>
시작 매듭(Start tag)요소의 내용(Element contents)끝 매듭(End tag)
<h1>문서의 주 제목</h1>
<p>이 글은 하나의 예문이다.<;/p>
<br>nonenone

참고 : 일부 HTML 요소는 <br> 요소와 같이 내용이 없는 요소도 있다. 이러한 요소를 빈 요소라고 하며 끝 매듭이 필요없다.

HTML 요소 겹치기

HTML 요소는 다른 요소와 겹치게 될 수 있다.
모든 HTML 문서의 HTML 요소는 겹쳐져 있다.

다음 예는 <html> 요소 안에 <body> 요소가 있고, <body> 요소 안에 <h1> 요소와 <p> 요소가 있다. 

  <!DOCTYPE html>
  <html>
      <body>
          <h1>글 머리</h1>
          <p>이 글은 하나의 문단이다.</p>
      </body>
  </html>

<html> 요소는 뿌리 요소이며, <html> 과 </html> 매듭 사이에 있는 내용이 HTML 문서임을 선언한다.

<html> 요소 안에는 <body> 요소가 있다.

<body> 요소는 <body>와 </body> 매듭 사이에 문서의 본문을 정의한다.

<body> 요소 안에는 <h1>과 <p> 요소가 있다.

<h1> 요소는 <h1> 과 </h1> 매듭 사이에 문서의 글 머리를 정의한다.

<p> 요소는 <p> 와 </p> 매듭 사이에 문단을 정의한다.

끝 매듭(End tag) 건너띄기

HTML 요소의 끝 매듭으로 마무리를 하지 않아도 올바르게 보여진다. 

  <html>
      <body>
      
          <p>이 글은 하나의 문단이다.
          <p>이 글은 또 다른 문단이다.

      </body>
  </html>

이 글은 하나의 문단이다.

이 글은 또 다른 문단이다.

이것에 의존하여 종료 태그를 잊어 버리면 예상치 못한 결과와 오류가 발생할 수 있다!

빈 HTML 요소

내용이 없는 HTML 요소를 빈 요소라고 한다. <br>태그는 줄 바꿈(break)을 선언하고, 끝 매듭이 없는 빈 요소이다 

<p>이 글은 <br>문단 안에서 줄 바꿈을 한다. </p>

문단 안에서 줄 바꿈을 한다.

HTML은 대-소문자를 구분하지 않는다.

HTML 표준에는 소문자 사용을 요구하지 않지만 W3C(World Wide Web Consortium) 는 HTML에서 소문자를 권장하며, XHTML과 같은 더 엄격한 문서 유형에서는 소문자 만을 사용한다.
