파일 경로( url )

URL(Uniform Resource Locator)은 인터넷에서 웹 문서, 그림, 동영상 등의 파일 위치를 가리키는 문자열이다.

기본 절대 주소 : URL의 구조 

  http://www.example.com:80/path/to/file_name.html

http 는 규약(protocol)이다. URL의 첫 부분은 브라우저가 어떤 규약을 사용해야 하는 지를 나타낸다. 규약은 컴퓨터 네트워크에서 데이터를 교환하거나 전송하기 위한 방법들의 집합이다. 일반적으로 웹 사이트들은 HTTP 규약이나 HTTP 규약의 보안 버전인 HTTPS 두 가지 중 하나를 사용한다 

  http://www.example.com:80/path/to/file_name.html

www.example.com 은 Domain 이름이다. 이것은 웹 서버의 주소이다. 웹 서버의 주소는직접 IP 주소를 사용하는 것도 가능하지만 IP 주소는 불편하기 때문에 웹에서 주로 사용되지는 않는다. 

  http://www.example.com:80/path/to/file_name.html

:80 은 port 이다. 이것은 기술적으로 웹 서버에서 파일에 접근하기 위해 사용하는 "관문( gate )"을 가리킨다. 만약 웹서버가 파일에 접근하기 위해 표준 HTTP port (HTTP를 위한 80, HTTPS를 위한 443) 를 사용한다면, port 번호는 보통 생략한다. 그렇지 않다면 포트 번호는 필수이다. 

  http://www.example.com:80/path/to/file_name.html

/path/to/file_name.html 은 웹서버에서 파일에 대한 경로이다. 웹서버상에서 물리적 파일 위치를 실제 물리적 경로를 나타내지는 않고, 웹 서버에서 추상화하여 보여준다. 

  http://www.example.com:80/path/to/file_name.html?key1=value1&key2=value2

?key1=value1&key2=value2 는 웹서버에 제공하는 추가 매개변수이다. 이 매개변수들은 & 기호로 구분된 키/값으로 짝을 이룬 목록이다. 웹 서버는 요구한 자료를 반환하기 전에 추가적인 작업을 할 때 이런 매개변수들을 사용한다. 각각의 웹서버는 매개변수들을 구분하는 자신의 규칙을 갖고 있다. 그리고, 특정한 웹서버가 매개변수를 다루는 지 알기 위한 유일한 방법은 웹서버 소유자에게 물어보는 것이다. 

  http://www.example.com:80/path/to/file_name.html?key1=value1&key2=value2#SomewhereInTheDocument

#SomewhereInTheDocument 는 자료 자체의 다른 부분에 대한 책갈피(anchor)이다. 즉, "책갈피" 지점에 위치된 내용을 보여주기 위해 브라우저에게 알려준다. 예를 들어, HTML 문서에서 브라우저는 책갈피가 정의한 곳으로 이동한다. 비디오나 오디오 문서에서, 브라우저는 책갈피가 나타내는 시간을 찾는다. 부분 식별자(fragment identifier)인 #뒤에 오는 부분은 서버에 보내지지 않는다.

상대 파일 경로

상대 파일 경로는 현재 페이지에 상대적인 파일을 가리킨다.

현재 웹의 루트에 있는 image 폴더의 파일 
  <img  src="/image/img_flower.jpg" alt="진달래">
현재 폴더 에 있는 image 폴더의 파일 
  <img  src="image/img_firstlove.jpg" alt="꽃말 첫사랑">
현재 폴더에서 한 수준 위에 있는 image 폴더의 파일 
  <img  src="../image/img_flwr.gif" alt="야생화">

가능한 상대 파일 경로를 사용하는 것이 좋다.
상대 파일 경로를 사용하면 웹 페이지가 현재 기본 URL에 묶이지 않는다.
모든 링크는 개발자의 컴퓨터 (localhost)와 현재 사용중인 도메인 및 향후 공용 도메인에서도 작동하도록 된다.
