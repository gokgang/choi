HTML 문서 작성 지침

깔끔하고 일관된 HTML 문서를 만들면 다른 사람이 문서를 더 쉽게 읽고 이해할 수 있다.

문서 형식 선언

문서의 첫 번째 줄에는 반드시 문서 형식을 선언한다.

<!DOCTYPE html>

소문자 요소 이름 쓰기

HTML은 요소 이름에 대문자와 소문자를 모두 쓸 수 있지만 소문자만 쓰는 것이 좋다.

대문자와 소문자의 이름을 썪어쓰면 보기에 좋지 않고, 소문자는 쓰기도 쉽고 깔끔하여 개발자들은 일반적으로 소문자를 쓴다. 

  <body>
    <p>이 글의 매듭과 같이 소문자를 쓰는 것이 좋다.</p>
  </body>

HTML 요소 닫기

HTML에서는 <p> 요소와 같은 요소는 닫지 않아도 되지만 모든 HTML 요소를 닫아 주는 것이 좋다. 

  <section>
    <p>이 글의 p 요소와 같이 닫아주는 것이 좋다. </p>
    <p>아래의 예와 같이 요소를 닫아주지 않는 것은 좋지 않다.</p>
  </section>

소문자 속성 이름 쓰기

HTML은 속성 이름에 대문자와 소문자를 썪어서 쓸 수 있다.
그러나 대문자와 소문자의 이름을 썪어쓰면 보기에 좋지 않고, 소문자는 쓰기도 쉽고 깔끔하여 개발자들은 일반적으로 소문자를 쓴다. 

  <a href="https://www.example.com/html/">HTML 배우기</a>

속성값 따옴표 묶기

HTML은 따옴표없이 속성값을 쓸 수 있지만, 따옴표로 묶으면 알기 쉬우며 속성값에 띄우기가 있으면 반드시 따옴표로 묶어야 하므로 많은 개발자들은 속성값을 따옴표로 묶는다. 

  <div class="header">

img 요소의 alt, width, height 반드시 지정

<img> 요소의 alt 속성은 어떤 이유로 그림을 보여줄 수 없는 경우 나타나므로 반드시 지정해야 한다. width 와 height 속성을 지정하면 브라우저가 파일을 내려받을 때 그림을 위한 공간을 미리 만들어 둔다. 

  <img src="flower.png"" alt="flower" style="width:128px; height:128px">

빈곳과 등호

HTML은 등호 앞뒤에 띄어쓰기를 할 수 있지만 빈곳이 없으면 읽기가 더 쉽고 개체를 더 잘 묶어준다. 

  <link rel="stylesheet" href="styles.css">

긴 줄 피하기

HTML 편집기를 사용할 때 HTML 코드를 읽기 위해 좌우 이동막대를 쓰는 것은 좋지 않으므로 너무 긴 줄은 피해야 한다. 많은 개발자들은 80자 이내로 작성한다.

빈 줄 및 들여 쓰기

이유없이 빈 줄, 띄어쓰기 또는 들여 쓰기를 하면 않된다.
알아보기 쉽게 빈 줄을 추가하여 논리적인 코드 묶음으로 구분하는 것이 좋다.
일반적으로 들여 쓰기는 두 개의 빈 곳을 둔다.
빈 곳은 탭 키를 사용하지 말고 띄어쓰기를 하도록 한다. 

  <body>
  
  <h1>유명한 도시</h1>

  <h2>Tokyo</h2>
  <p>도쿄는 도쿄 지역의 중심인 일본의 수도이다.
  세계에서 가장 인구가 많은 대도시 지역이다.
  이곳은 일본 정부와 황궁이 있는 곳이다.
  일본 황실의 고향이기도 하다.</p>

  </body>

  <table>
    <tr>
      <th>이름</th>
      <th>설 명</th>
    </tr>
    <tr>
      <td>A</td>
      <td>설명 A</td>
    </tr>
    <tr>
      <td>B</td>
      <td>설명 B</td>
    </tr>
  </table>

  <ul>
    <li>서울<li>
    <li>대구<li>
    <li>부산<li>
  </ul>

반드시 <title> 요소 지정

HTML에 <title> 요소는 반드시 필요하다.

<title> 요소의 내용은 Google과 같은 검색기 최적화(SEO)에 매우 중요하다!
<title> 요소는 검색기 알고리즘에서 검색 결과 문서를 나열할 순서를 결정한다.

<title> 요소는 :

  • 탐색기의 도구 모음에 제목으로 지정한다.
  • 탐색기의 즐겨찾기가 만들어질 때 제목으로 쓰여진다.
  • 검색기의 결과에 문서 재목으로 나타낸다.

따라서 틀림없고 뜻이 있는 제목으로 지정해야 한다.

  <title>HTML 문서 만들기에 지킬 것</title>

반드시 <html>과 <body> 매듭 사용

다음 예의 HTML 문서는 <html>과 <body>매듭없이 유효성을 검사한다.

  <!DOCTYPE html>
  <head>
    <title>Page Title</title>
  </head>

  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>

HTML 문서에 <html>과 <body> 매듭없이 만들어도 된다. 그러나 항상 <html>과 <body> 매듭을 쓰는 것이 좋다. HTML 문서에 <body> 매듭이 없으면 이전 탐색기에서 오류가 있을 수 있으며, <html>과 <body> 매듭이 없으면 DOM과 XML 끼리 충돌할 수 있다.

HTML 문서에 <head> 요소가 없어도 되지만 이전의 탐색기(Browsers)는 반드시 있어야 한다.

빈 HTML 요소 닫기

HTML에서 빈 요소를 닫는 것은 선택 사항이다.

  <meta charset="UTF-8">
  <meta charset="UTF-8" />

위의 두 가지 모두 쓸 수 있다. XML / XHTML 소프트웨어가 페이지에 접근할 것으로 예상하는 경우는 닫는 빗금 ( / )을 사용해야 한다. XML 및 XHTML에서는 반드시 필요하기 때문이다.

lang 속성 더하기

웹 페이지의 언어를 선언하려면 항상 <html> 매듭 안에 lang 속성을 포함해야 한다. 이는 검색기(Search Engines)와 탐색기(Browsers)를 지원하기 위한 것이다.

  <html lang="ko">

<meta> 요소

적절한 해석과 올바른 검색기 색인을 보장하려면 언어와 문자 인코딩을 모두 HTML 문서에서 가장 먼저 <meta charset="charset"> 요소를 정의해야 한다.

  <!DOCTYPE html>
  <html lang="ko">
  <head>
    <meta charset="UTF-8">
    <title>Page Title</title>
  </head>
  <body>
    ...
  </body>
  </html>

뷰포트 설정

뷰포트는 웹 페이지에서 사용자가 볼 수있는 영역이다. 장치에 따라 다르며 컴퓨터 화면보다 휴대폰에서 더 작다.
모든 웹 페이지에 다음의 <meta> 요소를 포함해야 한다.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

이것은 페이지의 크기 및 크기 조정을 제어하는 ​​방법에 대한 지침을 탐색기에게 제공한다.
이 width=device-width 부분은 장치의 화면 너비를 따르도록 페이지 너비를 설정한다.
이 initial-scale=1.0 부분은 탐색기가 페이지를 처음 로드할 때 초기 확대 / 축소 수준을 설정한다.

HTML 풀이글(comment)

짧은 풀이는 다음과 같이 한 줄에 작성한다.

  <!-- 이 글은 풀이글이다 -->

두 줄 이상의 풀이글은 다음과 같이 작성한다.

<!--
    이 글은 매우 긴 풀이글이다. 이 글은 매우 긴 풀이글이다.
    이 글은 매우 긴 풀이글이다. 이 글은 매우 긴 풀이글이다.
  -->

긴 주석은 두 개의 공백으로 들여 쓰면 알아보기가 더 좋다.

스타일 시트 사용

스타일 시트에 연결하기 위해 type 속성이 필요하지 않으며 간단한 구문을 사용한다.

  <link rel="stylesheet" href="styles.css">

CSS 규칙은 다음과 같이 짧게 압축하여 작성할 수 있다.

  p.intro {font-family:Verdana;font-size:16em;}

CSS 규칙은 여러 줄에 걸쳐 작성한다.

  body {
    background-color: lightgrey;
    font-family: "Arial Black", Helvetica, sans-serif;
    font-size: 16em;
    color: black;
  }
  • 선택자와 같은 줄에 여는 괄호를 둔다.
  • 여는 괄호 앞에 하나의 띄어쓰기를 한다.
  • 두 개의 들여 쓰기를 한다.
  • 마지막을 포함하여 각 속성-값 쌍의 뒤에 세미콜론을 둔다.
  • 값에 공백이 포함된 경우에만 값을 따옴표로 묶는다.
  • 앞에 띄움없이 새 줄에 닫는 괄호를 둔다.

외부 JavaScript 파일 불러오기

외부 JavaScript 파일을 불러오기 위해 type 속성이 필요하지 않으며 간단한 구문을 사용한다.

      <script  src="script.js">

JavaScript로 HTML 요소에 접근

JavaScript는 식별자(id)의 이름의 대·소문자를 구분하므로 다음 예는 다른 결과가 된다.

  <p id="Demo">JavaScript demo</p>
  <p id="demo">JavaScript demo</p>
  <script>
    document.getElementById("Demo").innerHTML = "Hello JavaScript Demo";
    document.getElementById("demo").innerHTML = "Hello 자바 스크립트 연습";
  </script>

파일 이름 소문자 사용

일부 웹 서버 (Apache, Unix)는 파일 이름에 대해 대·소문자를 구분한다.
다른 웹 서버 (Microsoft, IIS)는 대·소문자를 구분하지 않는다.
대문자와 소문자를 혼합하여 사용하는 경우 이를 알고 있어야 한다.
대·소문자를 구분하지 않는 서버에서 대·소문자를 구분하는 서버로 이동하면 작은 오류라도 웹을 망칠 수 있다!
이러한 문제를 방지하려면 항상 소문자를 파일 이름으로 사용하면 된다.

파일 확장자

HTML 파일의 확장자는 .htm 또는 .html 이어야 한다. .htm과 .html 파일 확장자에는 차이가 없다! 둘 다 웹 브라우저와 웹 서버에서 HTML로 처리된다.

CSS 파일의 확장자는 .css 이어야 한다.

JavaScript 파일의 확장자 는 .js 이어야 한다.

기본 파일 이름

"https://www.example.com/" 와 같이 URL의 끝에 파일 이름을 지정하지 않으면 , 서버는 "index.html", "index.htm" 또는 "default.html", "default.htm"과 같은 기본 파일 이름만 추가한다. 서버가 기본 파일 이름으로 "index.html"로만 구성된 경우 파일 이름은 "default.html"이 아니라 "index.html"이어야 한다. 그러나 서버는 둘 이상의 기본 파일 이름으로 구성할 수 있다. 일반적으로 원하는만큼 기본 파일 이름을 설정할 수 있다.
