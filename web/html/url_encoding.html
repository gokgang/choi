HTML Uniform Resource Locators

URL은 웹 주소의 또 다른 표현이다.

URL은 Domain 이름(예 : example.com) 또는 IP(Internet Protocol) 주소 (예 : 192.68.20.50)로 구성될 수 있다.
Domain 이름은 숫자로된 IP 주소 보다 기억하기 쉽기 때문에 대부분의 사람들은 도메인 이름을 사용한다.

Uniform Resource Locator

Web Browser는 URL을 사용하여 웹 서버에게 웹 문서를 요청한다. URL은 Web에서 문서 또는 자료의 주소를 지정하는 데 사용된다.

구문 규칙

scheme://prefix.domain:port/path/filename
  • scheme - 인터넷 서비스 유형. http 또는 https
  • prefix - 도메인 접두사. http의 기본값 www
  • domain - 도메인 이름
  • port - 호스트 의 포트 번호. http의 기본값 80
  • path - 서버 의 경로. 생략된 경우 루트 디렉토리
  • filename - 문서 또는 자료의 이름

일반적인 URL 용어

용어규약설명
http HyperText Transfer Protocol 암호화되지 않은 일반적인 web pages
https Secure HyperText Transfer Protocol 암호화된 보안 web pages
ftp File Transfer Protocol 파일 내려받기 또는 올리기
file   컴퓨터에 있는 파일

URL 부호화

URL은 인터넷을 통해서 ASCII 글자 모음 만을 사용하여 보낼 수 있다 . URL에 ASCII 모음 이외의 글자가 포함된 경우 URL을 바꾸어야 한다.
URL 부호화는 ASCII 글자가 아닌 자료를 인터넷을 통해 전송할 수 있는 형식으로 변환한다.
URL 부호화는 ASCII 글자가 아닌 글자를 "%"뒤에 16 진수로 대체한다.
URL은 띄어쓰기를 할 수 없다. URL 부호화는 일반적으로 띄어쓰기를 더하기 (+) 기호 또는 % 20으로 대체한다.
