head 요소

HTML <head> 요소는 브라우저가 식별할 수 있는 문서 정보( meta data )를 담을 수 있는 그릇과 같은 것이다.

참고 : HTML5 호환 브라우저는 <head> 가 없는 경우 자동으로 생성한다

  • <head> 안에 배치할 수 있는 요소
    • <title>
    • <base>
    • <link>
    • <style>
    • <meta>
    • <script>
    • <noscript>
    • <template>

<title> 요소

<title> 요소는 문서의 제목을 지정한다. 제목은 글자 전용이며 브라우저의 제목 표시줄이나 페이지의 탭에 표시된다.
<title> 요소는 검색 엔진 최적화 (SEO)에 매우 중요하다! 페이지 제목은 검색 엔진 알고리즘에서 검색 결과에 페이지를 나열할 때 순서를 결정하는 데 사용된다.

    <title> 요소
  • 브라우저 도구 모음에 제목을 지정한다.
  • 페이지가 즐겨 찾기에 추가될 때 페이지의 제목을 제공한다.
  • 검색 엔진 결과에 페이지 제목을 표시한다.
    • 이와 같은 이유로 제목은 정확하고 의미있게 만들어야 한다.

<style> 요소

<style> 요소는 하나의 HTML 문서에 고유의 양식을 지정하는 데 사용한다. 

  <style>
    body {background-color: powderblue;}
    h1 {color: red;}
    p {color: blue;}
  </style>

<link> 요소

<link> 요소는 현재 문서와 외부 파일 사이의 관계를 지정한다. <link> 매듭은 대부분 외부 CSS 파일에 연결하는 데 사용된다 

  <link  rel="stylesheet" href="mystyle.css">

즐겨찾기 아이콘( favicon ) 지정

  <link  rel="icon" href="favicon.ico">

  <link  rel="icon" href="favicon32.png">

  <link  rel="shortcut icon" href="https://www.example.org/img/favicon32.png">

대표적 favicon 파일 : favicon.ico 크기 : 32 × 32px

아이콘의 크기는 브라우저의 기본 글자 크기 16px에 맞추어지므로 크게 만들면 크기가 줄어들면서 그림이 손상될 수 있으므로 가급적 간단한 구조로 작게 만드는 것이 좋다. 32px로 하면 가끔 즐겨찾기 바로가기에 큰 아이콘을 필요로 할 때 도움이 된다. 그림 파일이면 ico 파일이 아니어도 되며, png 또는 gif 로 만들면 배경을 투명으로 하여 글자같이 보여지게 할 수 있다.

<meta> 요소

<meta> 요소는 일반적으로 문자 집합(charset), 문서 설명(description), 검색어(keywords), 문서 작성자(author), 화면 크기(viewport) 등을 지정하는 데 사용된다. <meta> 요소는 문서에 표시되지 않지만 브라우저가 내용을 표시하거나 문서를 다시 로드하는 방법, 검색 엔진의 검색어, 기타 웹 서비스에서 사용된다.

문자 집합(charset) 지정

<meta charset="UTF-8">

검색 엔진을 위한 검색어 지정

<meta name="keywords" content="HTML, CSS, JavaScript">

웹 페이지에 대한 설명 지정

<meta name="description" content="Free Web tutorials">

웹 페이지 작성자 지정

<meta name="author" content="이준석">

웹 문서 새로 ​​고침 시간 지정 30초

<meta http-equiv="refresh" content="30">

기기의 화면 크기(viewport) 지정

웹 문서를 사용자가 볼 수 있는 영역은 장치에 따라 다르며 컴퓨터 화면보다 휴대폰에서 더 작다. 따라서 모든 웹 문서는 다음의 요소를 포함해야 한다. 이것은 페이지의 크기 및 크기 조정을 제어하는 ​​방법에 대하여 브라우저에게 지침을 제공한다.
width=device-width 부분은 장치의 화면 너비를 따르도록 문서 너비를 장치에 따라 다르게 설정한다.
initial-scale=1.0 부분은 브라우저에서 페이지를 처음 로드할 때 초기의 확대 / 축소 수준을 설정한다.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

<base>

<base> 요소는 문서 안의 모든 상대 URL이 사용할 기준 URL을 지정한다. 문서에는 하나의 <base> 요소만 존재할 수 있다. 닫는 매듭은 없어야 한다.

<base> 요소의 속성

다음 특성 중 하나라도 지정한 경우, 상대 URL 속성을 사용한 어떤 요소보다 <base> 요소가 앞에 위치해야 한다.

href

문서 안에서 상대 URL이 사용할 기준 URL. 절대 및 상대 URL을 사용할 수 있다.

target

target 속성을 명시하지 않은 <a>, <area>, <form> 요소가 탐색을 실행했을 때, 그 결과를 보여줄 기본 브라우저 창을 지정한다. 속성값은 지시어나 개발자가 지정한 이름으로 설정한다.

    target 속성의 기본 지시어
  • _self (기본값) : 브라우저의 현재 창
  • _blank : 브라우저의 새로운 창
  • _parent : 브라우저의 현재 창의 상위 브라우저 창. 상위 창이 없으면 _self 와 동일
  • _top : 브라우저의 현재 창의 최상위 브라우저 창. 상위 창이 없으면 _self 와 동일

<base> 요소가 여러 개 존재하는 경우 첫 href 와 첫 target 만 사용하며 나머지는 모두 무시한다.
<a href="#some-id"> 처럼 문서의 책갈피( bookmark )를 가리키는 링크도 <base>를 사용해 처리하므로, 기준 URL 뒤에 책갈피를 붙인 주소로 HTTP 요청을 한다.

예 :

  1. <base href="https://example.com"> 로 지정하고
  2. <a href="#anchor">Anker</a>를 지정하면
  3. Anker 링크는 https://example.com/#anchor 를 가리키게 된다.

Open Graph

Open Graph 요소는 <base>를 인식하지 않으므로 항상 온전한 형태의 절대 URL을 지정해야 한다 

  <meta property="og:image" content="https://example.com/thumbnail.jpg">

  <base  href="http://www.example.com/page.html">
  <base  target="_blank">
  <base  target="_target" href="http://www.example.com/page.html">

<script> 요소

<script> 요소는 사용자 측 JavaScript를 정의하는 데 사용된다. 

  <script>
    function innerfunction () {
      document.getElementById("demo").innerHTML = "Hello JavaScript!";
    }
  </script>

      }
    </script>
  </head>
