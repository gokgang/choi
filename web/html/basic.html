HTML 문서의 기본 구조

  <!DOCTYPE html>
  <html>
  <head>
      <meta charset="UTF-8">
      <title>문서 제목</title>
  </head>
  <body>
  
      <h1>문서 제목</h1>
      <p>
          이 글은 하나의 문장이다.<br>
          <a href="https://www.w3.org/">W3C</a>는 World Wide Web Consortium 으로 웹 의 표준을 관장한다.
      </p>

      <img src="image/img_flower.png" width="150px">

  </body>
  </html>

<!DOCTYPE html>

이 문서가 html version 5 로 작성되었음을 선언한다.

HTML 문서는 대-소문자 구분을 하지 않지만, 가급적 소문자로 작성할 것을 권장한다.

<html>

html 문서의 처음 임을 선언한다.

<head>

문서의 글 머리(head)임을 선언한다. 여기에는 문서에 포함되는 각종 정보(meta data)와 문서의 제목(title), 스타일 시트(CSS), 스크립트(JavaScript) 등을 작성 또는 포함시킨다.

<meta charset="UTF-8">

이 문서가 유니코드 문자 집합 UTF-8로 작성되었음을 선언한다.
현재 웹 문서의 표준이 UTF-8이므로 작성하지 않아도 무방하지만 오래된 웹브라우저를 사용하는 사용자를 위해 선언해 주는 것이 좋다.

<title> ... </title>

문서의 제목을 선언한다

</head>

문서 글 머리의 끝임을 선언한다.

<body>

문서 몸체(body)가 되는 본문의 처음임을 선언한다.

<h1> ... </h1>

문서의 머리(Head) 부분에 해당하는 주 제목을 작성한다.
<h1>은 글자가 가장 크고 굵으므로 문서의 주 제목을 작성할 때 사용한다.
각 괄호로 된 기호 문자를 태그(Tag)라고 하며, 시작 태그와 빗금이 있는 끝 태그로 장식하고자 하는 문서의 내용을 하나의 매듭으로 묶는다. 매듭의 시작 태그와 끝 태그는 반드시 있어야 하지만 줄 바꿈 태그 <br> 과 같이 끝 태그가 필요없는 경우도 있다.

<p> ... </p>

<p> 매듭 안에 있는 글은 하나의 문단임를 선언한다. 이 매듭은 한 줄을 모두 차지하며, 한 줄이 넘는 경우에 자동으로 줄 바꿈이 되고, 끝 매듭이 나올 때까지 모두 하나의 문단을 이룬다.

<br>

<br> 태그는 줄 바꿈(break)이 필요한 경우 사용되며, 끝 태그가 필요없다. 물론 있어도 된다. 엄격한 문법을 지켜야 하는 DHTML 에서는 반드시 끝 태그가 있어야 한다.

<a href="https://www.w3.org/">W3C</a>

<a> ... </a> 태그는 다른 웹 문서를 연결한다.
연결 문서는 href 속성에 지정된다. 속성은 HTML 의 추가 정보를 지정한다.

<img src="image/img_flower.png" width="150px">

<img> 태그는 문서에 그림을 넣는다.
src="image/img_flower.png"는 그림이 있는 곳을 지정한다.
width="150px는 그림의 폭을 지정한다. 지정하지 않으면 원본 그대로 보여진다.

</body>

본문의 끝임을 선언한다.

</html>

HTML 문서의 끝임을 선언한다.

HTML 소스 보기

웹 브라우저(Chrome)에서 웹 문서를 마우스 오른쪽을 클릭하고 "페이지 소스 보기"를 선택하면, 문서에 사용된 소스 코드가 포함된 새로운 창이 열린다.
다른 웹 브라우저도 약간의 차이가 있지만 마찬가지이다.

HTML 요소 검사

HTML 요소 또는 빈 영역을 마우스 오른쪽 버튼으로 클릭하고 [검사] 또는 [요소 검사]를 선택하면, HTML과 CSS가 모두 표시되므로 어떤 요소로 구성되어 있는지 확인할 수 있다.
오른쪽을 향한 삼각형를 클릭하면 한 계단 아래의 요소가 나타나며, 아래를 향한 삼각형은 현재 열려있는 요소이다. 또는 요소 위에 마우스를 올려두면 소스보기의 해당 요소를 볼 수 있다.
아래쪽 패널에 있는 Style 탭에서는 HTML 또는 CSS를 즉석에서 확인하고 편집할 수도 있다.
